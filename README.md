# MAX Archive Betr 📸

⚠️ Важное предупреждение

Проект больше не работает. Wayback Machine перестала предоставлять архивные копии для домена i.oneme.ru и теперь возвращает ошибку:
This URL has been excluded from the Wayback Machine.
В связи с этим CDX API не выдаёт списков файлов, и галерея не может функционировать.

**MAX Archive Betr** — инструмент для просмотра и локального кэширования изображений из архива `i.oneme.ru`.
Проект получает список файлов через Wayback Machine (CDX API), автоматически скачивает изображения на ваш компьютер и предоставляет удобную веб-галерею с поиском и сортировкой.

Домен `i.oneme.ru` используется в мессенджере MAX как хранилище изображений. Этот архив позволяет исследовать сохранённые копии этих файлов.

---

## ✨ Основные возможности

- 📦 **Автоматическое обновление списка** — каждые 3600 секунд запрашиваются новые записи из Wayback Machine  
- ⬇️ **Локальное скачивание** — изображения сохраняются на диск (папка `downloads/`) с оригинальным параметром `r` в имени  
- 🖼 **Удобная галерея** — просмотр в виде сетки, lazy loading, пагинация  
- 🔍 **Поиск по параметру `r`** — минимальная длина запроса 3 символа  
- 📅 **Сортировка по дате архивации** (новые/старые сначала)  
- 🏷 **Метка "загружено"** для уже скачанных локально файлов  
- 🌐 **Встроенный веб-сервер** — галерея доступна по адресу `http://localhost:11984`

## 🔧 Принцип работы

1. **CDX API** — сервер запрашивает у Wayback Machine все сохранённые URL вида `i.oneme.ru/i*` с HTTP-статусом 200.  
2. **Генерация списка** — полученные записи сохраняются в `images.json` вместе с датой архивации и ссылкой на архивную копию.  
3. **Фоновое скачивание** — каждые 30 секунд выбирается самое новое ещё не скачанное изображение, и оно сохраняется в папку `downloads/`. Имя файла формируется из параметра `r` (например, `abc123.jpg`).  
4. **Веб-интерфейс** — галерея читает `images.json` и отображает карточки с превью (ссылка на архивную копию), оригинальной ссылкой и датой. Для локально сохранённых файлов добавляется метка `(загружено)`.

## 🖥 Руководство пользователя

### Интерфейс галереи

- **Сортировка** — кнопки «Новые сначала» / «Старые сначала» меняют порядок вывода.  
- **Поиск** — поле ввода ищет по параметру `r` в ссылке (минимум 3 символа).  
- **Пагинация** — по 20 карточек на странице, навигация сверху и снизу.  
- **Карточка изображения** — содержит превью (клик открывает полную версию в архиве), ссылку на оригинал и дату архивации. Если файл уже скачан локально, рядом с датой будет зелёная метка `(загружено)`.

### Данные

- `images.json` — обновляется автоматически при появлении новых записей в Wayback Machine.  
- `last_update.txt` — хранит дату последнего обновления списка.  
- `downloads/` — папка с локально сохранёнными изображениями.

## ⚠️ Известные проблемы

- **Первое скачивание может занять время** — из-за задержки между запросами (30 секунд) большие объёмы новых изображений будут обрабатываться постепенно.  
- **Ошибки при скачивании** — если архивная копия повреждена или недоступна, запись помечается как `failed` и повторные попытки не выполняются.   
- **GIF-анимации** — поддерживаются, но могут загружаться дольше из-за размера.
